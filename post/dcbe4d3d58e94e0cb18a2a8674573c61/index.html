使用 Coding 持续集成实现自动部署 Hexo

迁移到Hexo后，早期源文件和静态页面都是放在Github，速度一般。后来了解到Coding，于是把静态页面也放在了Coding，Coding的服务器在香港，速度不错，也不用备案。所以以Coding为主，Github作为备份，不过源码还是放在Github，编译部署后推送到Coding和Github。

Hexo之类的静态博客什么都好，不好的就是编译和部署。以前用动态博客，随时随地登录博客后台，写就完了。而静态博客需要搭建环境、需要编译静态文件、需要推送静态文件等等，本机配置好后不用再折腾，但是换一台电脑的话，有需要配置一次，挺麻烦的。

工作中了解到持续集成（Continuous Integration，简称CI），可以很好的解决这类问题，一切自动化，专心写博客（虽然现在一年才写一次）。

获得 Coding Token

个人令牌或者项目令牌，二选一即可。

个人令牌

登录Coding，右上角个人账户设置->访问令牌->新建令牌。

令牌描述随便，权限仅选择project:depot，确认，输入密码提交。

请保存好生成的令牌用户名和令牌密码，令牌密码只会显示一次，如果遗失的话，再重新生成即可。

最终产物：令牌用户名aISowiCldD，令牌密码c61a37b6a92742128e10c0871a027d6b333f8d10。

项目令牌

打开项目页面，选择项目设置->开发者选项->项目令牌->新建项目令牌。

令牌名称随意，过期时间看心情，代码仓库权限随意，建议选择指定代码仓库配置权限，然后勾选读写权限，点击新建提交。

点击查看密码获得刚刚创建的项目令牌密码。

最终产物：令牌用户名ptdi8j8hfa9m，令牌密码46e18c219a9b45b68e7728660d012437afb314b0。
